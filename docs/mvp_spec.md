# MVP 스펙 — 커플 관계 성장 앱

> 가트만 이론 기반 / 1차 릴리즈 범위

---

## 1. 제품 한 줄 정의

> "매일 일기를 쓰면 AI가 두 사람의 관계를 분석하고, 함께 성장할 퀘스트를 준다."

---

## 2. 포함 기능 (MVP)

| # | 기능 | 설명 |
|---|------|------|
| 1 | 커플 코드 연결 | 6자리 코드로 두 계정을 1:1 페어링 |
| 2 | 매일 일기 작성 | 5개 고정 질문에 답하는 구조화 일기 |
| 3 | 오늘의 리포트 | AI가 두 사람 일기를 비교 분석한 공동 리포트 |
| 4 | 오늘의 퀘스트 | 리포트 기반 퀘스트 1개 제시 + 완료 체크 |
| 5 | 커플 레벨 / 경험치 | 일기 작성·퀘스트 완료 시 XP 획득, 레벨업 |
| 6 | 관계 몬스터 | 침묵 괴물 / 비난 드래곤 2종만 |

### 2-1. 일기 5개 고정 질문

```
Q1. 오늘 파트너와 나눈 가장 기억에 남는 순간은?
Q2. 오늘 파트너에게 고마웠던 점이 있었나요?
Q3. 오늘 서로 어긋났거나 속상했던 일이 있었나요?
Q4. 지금 나의 감정 상태를 한 단어로 표현하면?
Q5. 내일 파트너에게 해주고 싶은 것이 있다면?
```

### 2-2. AI 리포트 구성

- 긍정/부정 신호 비율 (가트만 5:1 기준)
- 4가지 독 감지 여부 (비난·방어·경멸·담쌓기)
- 두 사람의 감정 온도 차이
- 오늘의 메시지 (공감 문장 1~2개)
- 관계 위험도: 🟢 안정 / 🟡 주의 / 🔴 위험

### 2-3. 퀘스트 트리거 규칙

| 감지 신호 | 제시 퀘스트 |
|----------|------------|
| 비난 패턴 | "부드러운 시작으로 대화 걸기" |
| 담쌓기 패턴 | "10분 함께 산책하기" |
| 긍정 비율 높음 | "오늘 칭찬 3번 전하기" |
| 기본 (신호 없음) | "5분 눈 마주치며 하루 이야기 나누기" |

### 2-4. 관계 몬스터

| 몬스터 | 출현 조건 | 처치 방법 |
|--------|----------|----------|
| 침묵 괴물 | 담쌓기 패턴 감지 또는 일기 미작성 2일 연속 | 퀘스트 완료 시 HP 감소 |
| 비난 드래곤 | 비난/경멸 패턴 감지 | 퀘스트 완료 시 HP 감소 |

몬스터 HP가 0이 되면 XP 보너스 + 커플 레벨 보상.

### 2-5. 커플 레벨 / 경험치

| 행동 | XP |
|------|-----|
| 일기 작성 (한 명) | +10 XP |
| 두 사람 모두 일기 작성 | 추가 +15 XP |
| 퀘스트 완료 | +20 XP |
| 몬스터 처치 | +50 XP |

레벨 구간: 1→2 (100 XP), 2→3 (250 XP), 이후 150 XP씩 증가.

## 4. 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 14 (App Router) + TypeScript | 풀스택 단일 레포, SSR/API Routes |
| 스타일 | Tailwind CSS | 빠른 프로토타이핑 |
| DB / 인증 | Supabase (PostgreSQL + Auth + RLS) | Auth 내장, 처음부터 클라우드 DB, 배포 그대로 사용 |
| 클라이언트 | @supabase/ssr | Next.js App Router 공식 권장 패키지 |
| AI 분석 | Anthropic API (Claude Haiku) | 일기 비교 분석 + 리포트 생성, MVP 단계 비용 절감 |

## 6. 화면 흐름도

```
[ 랜딩 / 로그인 ]
        │
        ├── 신규 가입 → [ 회원가입 ] → [ 커플 코드 화면 ]
        │                                     │
        │                    ┌────────────────┴────────────────┐
        │                    ▼                                 ▼
        │             [ 코드 생성 ]                      [ 코드 입력 ]
        │                    └────────────┬────────────────────┘
        │                                 ▼
        │                        [ 커플 연결 완료 ]
        │                                 │
        └── 기존 로그인 ──────────────────▼
                                  [ 홈 대시보드 ]
                                          │
                      ┌───────────────────┼───────────────────┐
                      ▼                   ▼                   ▼
              [ 일기 작성 ]        [ 오늘의 리포트 ]     [ 커플 현황 ]
                      │                   │                   │
              Q1~Q5 답변          두 일기 AI 분석       레벨 / XP 바
              저장 → XP +10       위험도 표시           몬스터 HP 바
                                  퀘스트 제시           퀘스트 완료 버튼
```

### 화면별 상세

#### 홈 대시보드
- 오늘 일기 작성 여부 (나 / 파트너) — 체크 뱃지로 표시
- 커플 레벨 + XP 프로그레스 바
- 현재 출현 중인 몬스터 (있으면 HP 바)
- "오늘의 리포트 보기" 버튼 (두 사람 모두 작성 시 활성화)

#### 일기 작성
- Q1 → Q2 → Q3 → Q4 → Q5 순서로 한 질문씩 표시
- 각 질문에 textarea, 다음 버튼으로 진행
- 마지막 질문 후 "저장" → XP +10 지급 → 홈으로 이동

#### 오늘의 리포트
- 두 사람 모두 일기 작성 완료 시 Claude API 호출
- 상단: 관계 위험도 뱃지 (🟢/🟡/🔴)
- 중단: 두 사람 긍정/부정 비율 비교 바
- 하단: 오늘의 메시지 + 퀘스트 카드
- 퀘스트 카드 안에 "완료했어요" 버튼 → XP +20

#### 커플 현황
- 레벨 숫자 + 레벨명 (예: Lv.3 "서로를 알아가는 중")
- XP 프로그레스 바 (다음 레벨까지 N XP)
- 몬스터 섹션: 출현 중 몬스터 카드 or "오늘은 평화롭습니다"
- 누적 퀘스트 완료 횟수

## 7. API 엔드포인트 (Next.js Route Handlers)

```
POST /api/auth/register          회원가입
POST /api/auth/login             로그인

POST /api/couple/create          커플 코드 생성
POST /api/couple/join            커플 코드 입력으로 연결

POST /api/diary                  일기 저장
GET  /api/diary?date=YYYY-MM-DD  특정 날 일기 조회

GET  /api/report?date=YYYY-MM-DD 리포트 조회 (없으면 AI 생성 트리거)
POST /api/quest/complete         퀘스트 완료 처리

GET  /api/couple/status          커플 레벨, XP, 몬스터 현황
```

## 8. Claude API 호출 설계

### 프롬프트 구조

```
[시스템]
당신은 가트만 심리학 기반 커플 관계 분석 전문가입니다.
두 사람의 일기를 읽고 다음을 JSON으로 반환하세요:
- riskLevel: "GREEN" | "YELLOW" | "RED"
- positivityRatio: number (소수점 1자리)
- horsemen: { criticism: bool, defensiveness: bool, contempt: bool, stonewalling: bool }
- message: string (공감 메시지 2문장, 한국어)
- quest: string (오늘의 퀘스트 한 줄)

[유저]
[파트너 A 일기]
Q1: {a.q1}
Q2: {a.q2}
Q3: {a.q3}
Q4: {a.q4}
Q5: {a.q5}

[파트너 B 일기]
Q1: {b.q1}
...
```

- 모델: `claude-haiku-4-5-20251001` (비용 절감, MVP 단계)
- 두 사람 중 한 명만 작성한 경우: 리포트 생성 불가, 대기 메시지 표시
- 응답 파싱 실패 시: 기본 퀘스트 + "분석 중 오류" 메시지로 폴백
