# 일기 질문 설계 — EFT 환기 기법 기반

## 설계 원칙

각 질문은 다음 3단 구조로 응답자를 안내한다:

1. **오늘의 사건** — 가벼운 진입점 (기억에 남은 순간, 표면 사실)
2. **표면 감정** — 이차 정서 (짜증, 지침, 무관심 등 방어적 감정)
3. **진짜 마음** — 일차 정서 + 갈망 (외로움, 두려움, 인정 욕구 등)

모든 질문은:
- **환기적** — 정보 수집이 아니라 내면으로 데려감
- **비난 비유도** — "상대가 ~했다" 구조를 자연스럽게 억제
- **내면 집중** — "나는 ~느꼈다" 형식을 유도

---

## 5개 질문 상세 설계

---

### Q1. 오늘의 기억 질문

**질문:**
> 오늘 파트너와의 관계에서 가장 마음에 남은 순간은 언제였나요?
> 그때 몸 어디에서, 어떤 느낌이 들었나요?

| 항목 | 내용 |
|---|---|
| **질문 의도** | 오늘의 관계 온도를 구체적인 기억 + 신체 감각으로 착지시킨다. 추상적인 "좋았다/나빴다"가 아니라 "어디에서 느꼈는가"로 정서 경험을 현존화한다. |
| **유도하는 일차 정서** | 따뜻함·연결감 (긍정 시) / 고립감·외로움·거절감 (부정 시) |
| **유도하는 갈망** | 함께함, 보여지는 것(being seen), 연결의 확인 |
| **사용된 EFT 기법** | 환기적 질문 + 신체 감각 접근(Somatic Tracking) |
| **분석 AI가 추출할 신호** | - 기억의 유무 (공백 → 단절 신호) / - 신체 묘사 유무 (있으면 일차 정서 접근 중) / - 기억의 정서가 따뜻함인지 불편함인지 / - 파트너 묘사의 구체성 (사랑의 지도 지표) |

---

### Q2. 삼킨 감정 질문

**질문:**
> 오늘 표현하지 못하고 삼킨 감정이 있다면 무엇인가요?
> 왜 그 감정을 말하지 못했을까요?

| 항목 | 내용 |
|---|---|
| **질문 의도** | 이차 정서(표면)와 그 억압의 이유를 함께 끌어낸다. "왜 말하지 못했는가"가 일차 정서와 안전 이슈로 이어지는 핵심 통로다. |
| **유도하는 일차 정서** | 두려움(거절/갈등), 수치심(예민하다는 소리 들을까봐), 유기 불안 |
| **유도하는 갈망** | 안전한 표현 공간 갈망, 들어주기를 바라는 욕구 |
| **사용된 EFT 기법** | 환기적 질문 + 공감적 추측 (억압 이유 탐색) |
| **분석 AI가 추출할 신호** | - 억압된 감정의 종류 (이차 → 일차 추론 트리거) / - 억압 이유 패턴 (두려움? 포기? 익숙함?) / - 반복 억압 여부 (history 대조 시 사용) / - "어차피" 언급 → 담쌓기 전조 신호 |

---

### Q3. 듣고 싶었던 말 질문

**질문:**
> 그 감정 밑에, 사실은 파트너에게 듣고 싶었던 한 마디가 있었다면?
> 그 말을 들었다면 어떤 느낌이 들었을 것 같아요?

| 항목 | 내용 |
|---|---|
| **질문 의도** | 애착 갈망을 직접 언어화하도록 돕는다. "듣고 싶었던 말"은 갈망의 가장 순수한 표현이다. "들었다면 어떤 느낌"은 그 갈망이 충족될 때의 안전감·연결감을 체험하게 한다. |
| **유도하는 일차 정서** | 인정 갈망, 확인 욕구, 보살핌받는 느낌 |
| **유도하는 갈망** | 수용·가치 인정·"내 편"이라는 확신 |
| **사용된 EFT 기법** | 애착 씨뿌리기 (갈망 직접 환기) + 반영 |
| **분석 AI가 추출할 신호** | - "듣고 싶었던 말" → 직접 attachmentLongings 필드로 매핑 / - 들었다면의 감정 → 일차 정서 후보 / - 파트너가 그 말을 했던 적 있는지 대조 (history 활용) / - 갈망의 구체성 (구체적일수록 관계에 투자된 상태) |

---

### Q4. 보이는 느낌 질문

**질문:**
> 오늘 파트너가 당신을 "봐준다"고 느낀 순간이 있었나요?
> 없었다면, 어떤 순간이었기를 바랐나요?

| 항목 | 내용 |
|---|---|
| **질문 의도** | "보여지는 경험(being seen)"은 EFT에서 가장 근본적인 애착 욕구다. 있고 없음을 모두 물어봄으로써 기대와 현실의 간극을 직접 측정한다. |
| **유도하는 일차 정서** | 연결감·온기 (있었을 때) / 고립·무가치감·비가시성 (없었을 때) |
| **유도하는 갈망** | "내 존재를 알아봐 달라"는 갈망, 우선순위가 되고 싶은 욕구 |
| **사용된 EFT 기법** | 환기적 질문 + 공감적 추측 (없었다면 어떤 순간을 원했는가) |
| **분석 AI가 추출할 신호** | - "없었다" 응답 빈도 (history 대조 → 반복될수록 위험) / - "바랐던 순간" 묘사의 구체성 → 갈망 정밀도 / - 상대방의 작은 행동에 대한 민감성 (높으면 불안정 애착 신호) / - 긍정 경험 묘사 → 입금 신호로 분류 |

---

### Q5. 관계 온도 질문

**질문:**
> 오늘 우리 관계의 온도는 몇 점인가요? (1~10)
> 그 점수가 된 이유는 무엇인가요?

| 항목 | 내용 |
|---|---|
| **질문 의도** | 정량적 지표(온도 수치)와 정성적 이유를 함께 수집한다. 수치는 두 사람의 온도차 감지에, 이유는 그날의 핵심 정서 신호 추출에 사용된다. |
| **유도하는 일차 정서** | 이유 설명 과정에서 핵심 감정이 자연스럽게 표면화됨 |
| **유도하는 갈망** | 이유에 따라 다양 — 온도가 낮을수록 갈망이 명확하게 드러남 |
| **사용된 EFT 기법** | 반영 (수치를 정서로 번역) + 환기적 질문 (이유 탐색) |
| **분석 AI가 추출할 신호** | - temperatureGap (두 사람 수치 차이) → 3 이상이면 YELLOW 경고 / - 이유에서 이차 정서 추출 → 일차 정서 추론 / - history와 대조 → 온도 추세 (하락 추세가 5일 이상이면 경고) / - "그냥", "모르겠어" 응답 → 정서적 마비 신호 검토 |

---

## 전체 질문 흐름 요약

```
Q1 (기억)  → 오늘 관계의 구체적 사건 착지
     ↓
Q2 (삼킨 감정) → 억압된 이차 정서 + 억압 이유 탐색
     ↓
Q3 (듣고 싶었던 말) → 애착 갈망 직접 언어화
     ↓
Q4 (보이는 느낌) → "존재 인정" 욕구 측정
     ↓
Q5 (온도 + 이유) → 정량 + 정성 통합 마무리
```

Q1→Q2: 사건에서 감정으로
Q2→Q3: 억압에서 갈망으로
Q3→Q4: 갈망에서 관계 욕구로
Q4→Q5: 순간에서 전체 온도로

---

## 분석 AI 주의사항

- Q3의 "듣고 싶었던 말"은 그대로 파트너에게 전달 **금지** — 번역 필수
- Q2의 "삼킨 감정"은 일기 원문 절대 공유 **금지**
- Q5의 온도 수치가 7 이상이더라도 이유에서 부정 신호가 나오면 YELLOW 이상으로 처리
- 모든 질문에서 "비어 있는 응답"은 없다 — 빈칸은 그 자체로 정서적 신호(회피, 마비, 포기)
